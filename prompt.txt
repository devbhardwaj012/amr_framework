## THE REAL-WORLD PROBLEM BEING SOLVED

A warehouse has 5 autonomous robots (AMRs) driving around on a 20×20 grid, picking up and delivering goods. Three hard problems must be solved simultaneously in real time:

1. **Coordination** — Robots must not collide, tasks must be assigned to the best available robot
2. **Energy** — Robots must recharge before their battery dies without unnecessarily leaving tasks
3. **Security** — The 5G network these robots communicate over gets attacked (PACKET_FLOOD, JAMMING, DATA_EXFIL, REPLAY_ATTACK) and threats must be detected and isolated instantly

Three AI agents — each a specialist — cooperate over an async event bus to solve these. Every agent uses a fast algorithmic layer (rule engine / Q-learning / Isolation Forest) for real-time decisions, and an LLM layer (Groq/LLaMA3-70B) for ambiguous situations, natural language reporting, and intelligent arbitration.

     │  engine)  │  │Rules+   │ │ QL+LLM   │ │ IF+LLM   │

                                     │

                         │   (async pub/sub)       │
                         └───────────┬────────────┘
                                     │
                         ┌───────────▼────────────┐
                         │       State Store       │
                         │  (memory dict + SQLite) │
                         └────────────────────────┘
```

**Every tick, this happens in order:**
1. Simulator moves AMRs, advances tasks, spawns new tasks
2. Simulator fires `TICK` event with fresh `SystemSnapshot`
3. All 3 agents receive snapshot simultaneously (async)
4. Each agent makes decisions, calls `store.update_*()` methods
5. Agents publish result events (`TASK_ASSIGNED`, `BATTERY_LOW`, `AMR_QUARANTINED`, etc.)
6. Other agents react to those events
7. Dashboard reads `store.get_snapshot()` and re-renders

**Golden rule**: Agents communicate ONLY via the event bus. They NEVER import or call each other directly.

## BASE AGENT — WHAT EVERY AGENT INHERITS

```python
class BaseAgent(ABC):
    # Constructor — takes bus, store, config dict
    def __init__(self, name: str, bus, store, config: dict = None)

    # ── Override these in your agent ────────────────────────────
    async def setup(self) -> None:          # subscribe events, init models
        ...  # abstract — MUST implement

    async def on_tick(self, snapshot: SystemSnapshot) -> None:
        ...  # abstract — MUST implement — called every tick

    async def run(self) -> None:            # optional background loop
        pass                                # default is no-op

    async def teardown(self) -> None:       # cleanup on shutdown
        self._running = False               # default logs teardown

    # ── Available to all agents ──────────────────────────────────
    await self.emit(EventType.X, payload_dict, severity=AlertSeverity.INFO)
    await self.log("message", AlertSeverity.INFO, metadata=None)
    self.get_stats() -> dict                # returns base perf counters

    # ── Auto-tracked counters ────────────────────────────────────
    self._events_received:  int   # incremented manually in handlers
    self._events_published: int   # auto-incremented by emit()
    self._decisions_made:   int   # increment when making a decision
    self._tick_count:       int   # auto-incremented by on_tick loop

    # ── Available references ─────────────────────────────────────
    self.bus     # EventBus
    self.store   # StateStore
    self.config  # flat dict — use self.config.get("key", default)
    self.logger  # Python logger named f"agent.{name}"
    self.name    # str
```

**How TICK events reach on_tick():** `BaseAgent.start()` subscribes `self._handle_tick` to `EventType.TICK`. `_handle_tick` calls `self.store.get_snapshot(tick)` then calls `self.on_tick(snapshot)`. You never subscribe to TICK yourself.

## AGENT 1: COORDINATOR (`agents/coordinator_agent.py` — 452 lines)

**Responsibility:** Collision avoidance, task assignment, fleet rebalancing

### Collision Detection (every tick)
O(n²) pairwise check on all non-charging, non-quarantined AMRs:
- `distance < 2.0 units` → PATH_NEGOTIATION: one AMR yields (status → IDLE for 1 tick)
- `distance < 1.0 unit` → COLLISION_WARNING: emergency stop (both → IDLE)

Who yields is determined by this priority chain:
1. Lower task priority yields
2. If equal priority AND LLM available → `_llm_negotiate_collision()`
3. If equal priority AND no LLM → higher battery yields (conserve low-battery AMR)

### Task Assignment (every tick, when pending tasks exist)
Only assigns to AMRs with `status == IDLE`, `not is_compromised`, `battery > 10%`

Scoring formula:
```python
score = max(0, 10 - distance_to_task) + (battery / 10.0) + (5.0 if IDLE else 0.0)
```
If top 2 scores are within `2.0` pts AND LLM available → `_llm_pick_assignee()`

### Fleet Rebalancing (every tick, when ≥ 2 idle AMRs)
- Idle AMRs within 3.0 units of each other → spread them apart
- If >50% of fleet is idle AND LLM available AND ≥3 idle AMRs → `_llm_fleet_rebalance()`

## AGENT 2: ENERGY AGENT (`agents/energy_agent.py` — 456 lines)

**Responsibility:** Prevent battery depletion via Q-learning

### Q-Table
```
Shape: (4, 3, 2, 3)  →  battery_bins × dist_bins × queue_bins × actions
Total: 72 state-action values
```

**State encoding:**
```python
battery_bin: 0 = critical (≤10%)  # NOTE: this overrides Q-table — force charge
             1 = low (10-25%)
             2 = medium (25-60%)
             3 = high (>60%)

dist_bin:    0 = close  (< 5 units to nearest free station)
             1 = medium (5-12 units)
             2 = far    (> 12 units)

queue_bin:   0 = empty (no pending tasks)
             1 = busy  (tasks in queue)
```

**Actions:**
```python
0 = ACTION_CONTINUE       # keep working
1 = ACTION_GO_CHARGE      # head to nearest free station
2 = ACTION_REDUCE_SPEED   # slow down to conserve battery
```

**Reward structure:**
```python
Critical battery reached:         -20  (big penalty — we failed)
Proactive charge (battery low):   +5   (good — caught it early)
Premature charge (battery high):  -3   (wasteful)
Continue with healthy battery:    +2   (good efficiency)
Low battery + chose CONTINUE:     -5   (risky decision)
REDUCE_SPEED:                     +1   (mild conservation)
```

**Pre-seeded Q-table (starts smart, not random):**
```python
Battery bin 0 (critical): [-20,  15,  -5]  # strongly prefer GO_CHARGE
Battery bin 1 (low):      [ -5,   8,   2]  # prefer GO_CHARGE
Battery bin 2 (medium):   [  8,  -3,   3]  # prefer CONTINUE
Battery bin 3 (high):     [ 10,  -8,   0]  # strongly prefer CONTINUE
```

**Bellman update (every tick per AMR):**
```python
new_q = current_q + α * (reward + γ * max_next_q - current_q)
# α=0.1, γ=0.9, ε=0.15 (greedy 85% of the time)
```

**Decision pipeline per AMR per tick:**
1. If battery ≤ 10% → force charge regardless of Q-table
2. Compute state (battery_bin, dist_bin, queue_bin)
3. ε-greedy action selection from Q-table
4. Apply action, observe reward
5. Bellman update on Q-table

## AGENT 3: SENTINEL (`agents/sentinel_agent.py` — 544 lines)

**Responsibility:** 5G network security via Isolation Forest + LLM incident reporting

### Attack Simulation
Attack probability: `0.005` (0.5%) per AMR per tick. Duration: 5-12 ticks.

Normal traffic baseline:
```
packet_size:     ~512 bytes (Gaussian, σ=30)
latency_ms:      ~5.0 ms (Gaussian, σ=0.5)
signal_strength: ~-70.0 dBm (Gaussian, σ=2.0)
packet_loss:     0.0–3% (uniform)
```

Attack signatures:
```
PACKET_FLOOD  → size: 40-80 bytes,  latency: ~100ms,       signal: normal, loss: <5%
JAMMING       → size: normal,       latency: 10-20ms,      signal: -108 to -115dBm, loss: 70-99%
DATA_EXFIL    → size: 40-65 KB,    latency: ~2.5ms,       signal: normal, loss: <2%
REPLAY_ATTACK → size: normal,      latency: 7.5-12ms,     signal: -98 to -105dBm, loss: 50-90%
```

### Isolation Forest Pipeline
```
1. Collect packets in rolling buffer (maxlen=100) per AMR
2. After 30+ NORMAL packets → fit IsolationForest(contamination=0.05, n_estimators=100)
3. StandardScaler applied to 4-feature vector [size, latency, signal, loss]
4. Score each new packet:
   Normal traffic:  -0.40 to -0.50  (within expected distribution)
   Attack traffic:  -0.60 to -0.80  (outliers flagged)
5. score < -0.55 → WARNING alert (log + emit INTRUSION_ALERT warning)
6. score < -0.65 → QUARANTINE + LLM incident report
```

### Quarantine Flow
```python

## LLM INTEGRATION — EXACT PATTERN USED EVERYWHERE

Every single LLM call in the codebase follows this exact pattern. Do NOT deviate from it:

```python
import json, os, logging
logger = logging.getLogger(__name__)

# In __init__:
self._groq_client = None
self._llm_enabled = bool(os.getenv("GROQ_API_KEY"))
self._llm_model   = os.getenv("LLM_MODEL", "llama3-70b-8192")
self._llm_calls   = 0     # track for dashboard

# In setup():
if self._llm_enabled:
    try:
        from groq import Groq
        self._groq_client = Groq(api_key=os.getenv("GROQ_API_KEY"))
    except ImportError:
        logger.warning("groq not installed — pip install groq")
        self._groq_client = None
        self._llm_enabled = False

# An LLM call method:
async def _llm_your_decision(self, ...context...) -> dict:
    prompt = f"""You are an AI warehouse fleet manager.

[Context: inject relevant data here as f-string]

Respond in EXACTLY this JSON format (no other text):
{{"key": "<value>", "reason": "<one sentence>"}}"""

    try:
        response = self._groq_client.chat.completions.create(
            model=self._llm_model,
            messages=[{"role": "user", "content": prompt}],
            max_tokens=100,       # keep low for JSON responses
            temperature=0.1,      # near-deterministic for decisions
        )
        self._llm_calls += 1
        text = response.choices[0].message.content.strip()
        # ALWAYS strip markdown fences — Groq sometimes wraps in ```json
        text = text.replace("```json", "").replace("```", "").strip()
        return json.loads(text)
    except Exception as e:
        logger.warning(f"[YourAgent] LLM call failed: {e}")
        # ALWAYS return a valid fallback — never propagate LLM exceptions
        return {"key": "fallback_value", "reason": "LLM unavailable"}
```

**Rules for LLM calls:**
1. Always request JSON output in the prompt
2. Always strip markdown fences before `json.loads()`
3. Always have an `except Exception` fallback that returns valid data
4. Non-blocking LLM calls (like incident reports) use `asyncio.create_task()`
5. Increment `self._llm_calls` on every successful API call
6. Keep `max_tokens` low for JSON responses (80-150), higher for narratives (100-250)
7. Use `temperature=0.1` for decisions, `temperature=0.3` for summaries/narratives

## HOW THE DASHBOARD WORKS

The dashboard is Streamlit with 4 pages. The orchestrator runs in a **background daemon thread** started by `@st.cache_resource` on first load.

```python
# dashboard/app.py
@st.cache_resource
def get_orchestrator():
    """Starts orchestrator once, cached for the entire Streamlit session."""
    loop = asyncio.new_event_loop()
    orch = Orchestrator()
    
    def run_loop():
        asyncio.set_event_loop(loop)
        loop.run_until_complete(orch.start())
        loop.run_forever()
    
    thread = threading.Thread(target=run_loop, daemon=True)
    thread.start()
    time.sleep(2)  # wait for agents to start
    return orch, loop

## IMPORTANT CONVENTIONS & GOTCHAS

1. **`NetworkPacket.packet_loss`** — the field is `packet_loss`, NOT `packet_loss_rate`. Using the wrong name breaks all Sentinel tests silently.

2. **All state-writing methods on StateStore are `async`** — always `await` them. Reading methods (`get_amr`, `get_all_amrs`, `get_snapshot`) are sync.

3. **Never import agents from each other.** All inter-agent communication happens exclusively through the event bus.

4. **The `config` inside agents is a flat dict** (`self.config.get("num_amrs", 5)`). The typed `SystemConfig` is only in Orchestrator and ConfigManager.

5. **`SystemSnapshot` is immutable** — it's a point-in-time copy. Modifying snapshot fields does nothing to real state. Use `store.update_*()` methods.

6. **LLM calls must be non-blocking when triggered by events** (e.g. quarantine incidents). Use `asyncio.create_task(self._llm_method(...))` — don't `await` them inline.

7. **The Simulator creates AMRs** — never create `AMRState` objects in agents. Agents only modify existing AMRs via `StateStore`.

8. **`self._decisions_made += 1`** should be incremented every time an agent makes a meaningful decision. This counter drives dashboard agent comparison charts.

9. **`get_stats()` in agents** — always call `super().get_stats()` first, then extend the dict. The base class tracks events_received, events_published, decisions_made, uptime, ticks_processed.

10. **Tests use `loop.run_until_complete()`** not `asyncio.run()` — the test runner passes its own event loop to each test function.
